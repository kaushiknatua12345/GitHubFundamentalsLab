# GitHub Copilot Training Exercises

These exercises are for product managers, technical writers, and other non-technical learners.

The goal is to practice GitHub and GitHub Copilot in simple workshop tasks without writing code.

## How to use this guide

1. Open `shared-context.md` to understand the sample project.
2. Use `requirement-assistant.md` to turn a plain request into a project plan.
3. Use Ask mode to ask questions about the frontend app.
4. Use Plan mode to organize work before making any changes.
5. Use Agent mode to create documentation drafts with Copilot.

## Workshop rules

- Keep every exercise small.
- Use plain language.
- Do not ask for full application changes.
- Review Copilot output before accepting it.
- Follow the reusable AI workflow files in `.ai-workflows`.

## Practice Work

### Exercise 1: Understand the frontend app

**Goal:** Learn what the sample app does in plain words.

**Task:**
- Open the app in `front end app`.
- Ask Copilot to explain the app for a product manager.
- Ask Copilot to explain the app for a technical writer.

**Use Copilot in:** Ask mode

**Expected outcome:**
- Two short explanations in simple language.

### Exercise 2: Create a project plan

**Goal:** Practice turning an idea into a small plan.

**Task:**
- Use `requirement-assistant.md`.
- Ask for a project plan for a simple employee registration training exercise.
- Include steps, owners, and a short delivery sequence.

**Use Copilot in:** Plan mode

**Expected outcome:**
- A clear, non-technical project plan.

### Exercise 3: Write a user manual draft

**Goal:** Practice creating user-facing documentation.

**Task:**
- Use Agent mode.
- Ask Copilot to draft a short user manual for the sample frontend app.
- Keep the tone simple and friendly.

**Use Copilot in:** Agent mode

**Expected outcome:**
- A short user manual draft that explains what a learner sees on the screen.

### Exercise 4: Write a technical document draft

**Goal:** Practice writing internal technical documentation in simple terms.

**Task:**
- Use Agent mode.
- Ask Copilot to draft a technical overview for the sample app.
- Include what the app does, where the front end lives, and how the in-memory data behaves.

**Use Copilot in:** Agent mode

**Expected outcome:**
- A short technical document draft for a writer to refine.

### Exercise 5: Branch and merge practice

**Goal:** Practice standard GitHub collaboration steps.

**Task:**
- Create a branch from `main` named `training/docs-update`.
- Add a small note to a markdown file.
- Use a pull request to merge the branch back into `main`.

**Use Copilot in:** Ask mode for guidance, Plan mode to sequence the steps

**Expected outcome:**
- A branch, a change, and a merge back to `main`.

### Exercise 6: Review a written change

**Goal:** Practice checking work before merge.

**Task:**
- Ask Copilot to review the markdown change for clarity.
- Check whether the wording is easy for non-technical readers.

**Use Copilot in:** Ask mode

**Expected outcome:**
- A short review with suggested improvements.

### Exercise 7: Prepare a release note draft

**Goal:** Practice writing simple release notes.

**Task:**
- Ask Copilot to draft release notes for the training project.
- Keep it short and easy to understand.

**Use Copilot in:** Agent mode

**Expected outcome:**
- A plain-language release note draft.

## Suggested learning path

1. Do Exercise 1.
2. Do Exercise 2.
3. Do Exercise 3.
4. Do Exercise 4.
5. Do Exercise 5.
6. Finish with Exercises 6 and 7.

## Example prompts

### Ask mode

"Explain the sample frontend app to a product manager in two sentences."

### Plan mode

"Create a simple project plan for a training exercise about the sample frontend app."

### Agent mode

"Draft a short user manual for the sample frontend app using plain language."

## Success criteria

- Learners can explain the app in plain language.
- Learners can create a simple project plan.
- Learners can draft user and technical documentation.
- Learners can create a branch and merge it back to main.
- Learners can use Ask, Plan, and Agent mode correctly.
